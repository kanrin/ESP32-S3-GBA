# KARPATHY 编码准则（必须严格遵守）
你在所有编码任务中，必须遵守以下 4 条核心原则：

## 1. 先思考，再编码
- 先明确说明你的所有假设，不擅自猜测需求
- 需求不明确时，必须提问澄清，不自己脑补
- 先给出多种方案对比，再开始写代码
- 遇到困惑立即停止并确认，不强行继续
- 对过度复杂的需求主动提出简化建议

## 2. 极简至上
- 只写解决当前需求的最少代码
- 不添加任何多余功能、抽象、扩展性
- 不做没必要的异常处理
- 不过度设计、不封装未来可能用到的逻辑
- 代码越短越好，能一行实现绝不写多行

## 3. 精准修改，不改动无关内容
- 只修改与当前需求直接相关的代码
- 不重构、不优化、不清理不相关的代码
- 不修改原有注释、不删除原有注释
- 完全匹配项目原有代码风格
- 不清理之前就存在的无用代码，除非明确要求

## 4. 目标驱动，可验证
- 开始编码前先定义清晰的成功标准
- 修复bug：先复现 → 修复 → 验证通过
- 做功能：先明确效果 → 实现 → 验证效果
- 每一步都要有验证环节
- 多任务分步执行，每步确认无误再继续

所有代码操作必须严格遵守以上规则！
